# UX Epics & Task Decomposition (2026-07-13)

**Status:** planning only — nothing starts until the in-flight master-spec agent finishes
(owner decision). Companion docs: [`../design/ux-vision-2026-07-13.md`](../design/ux-vision-2026-07-13.md) (rationale), [`../design/ux-master-spec-delta-2026-07-13.md`](../design/ux-master-spec-delta-2026-07-13.md)
(schema/contract changes; epics marked Δ depend on its approval). Scale calibration: all
acceptance criteria assume the owner's real scale — **30+ atlases, 5000+ sprites**.

Legend: **[gui]** apps/gui only · **[core]** packer/ core work · **[engine]** needs issue+PR in
engine repo · **[Δ]** blocked on delta-spec approval · **[F2]** needs typed-operation layer.

---

## UX-G0 · Bench fixture & perf harness  *(do first — everything is measured against it)*
Goal: a generated project at owner scale + repeatable measurements.
1. Fixture generator: script producing 30 atlases / 5000+ sprites (sizes/folders distribution
   like a real game). Accept: `scripts/gen_bench_project` creates it deterministically. [core]
2. Perf probes: frame-time + action-latency logging behind a dev flag; `--shot`-style headless
   run reports timings. Accept: budgets are numbers in CI output, not vibes. [gui]
3. Budgets wired as regression checks: interaction <100 ms, pack UI never blocked, refresh
   non-blocking. Accept: exceeding budget fails the check on the fixture. [gui][core]

## UX-A · Paper cuts (wave A — first implementation batch after unblock)
Goal: kill daily friction; no schema changes, no conflicts with master-spec work.
1. Tree filter (Ctrl+F), project-wide, matches under their atlases; <100 ms on fixture. [gui]
2. Folder collapse/expand (persistent per session) + scrollable full left panel. [gui]
3. Keyboard navigation: Up/Down/Home/End/Enter/F2-rename, focus ring, list focus model. [gui]
4. Session persistence: window geometry, UI scale, zoom/camera, recent projects, Open Recent
   menu, reopen-last-project option (app-state file, never in project). [gui]
5. Undo hygiene: undo/redo preserves selection and per-atlas previews of untouched atlases. [gui]
6. Menu completion: Pack in menubar (Atlas menu), tooltips with shortcuts everywhere. [gui]
7. Row utilities: sort (4 keys × 2 directions + "⚠ on top" checkbox, view state), Show in
   Explorer, Copy name, double-click canvas region ↔ zoom-to. [gui]
8. Sprite/atlas thumbnails in rows + hover 1:1 preview; async thumb cache (LRU, budget-bounded,
   fixture-proof). [gui]
9. Async refresh: hash-diff scan off-thread, progress + cancel; adaptive cache (no 32-dir cap);
   non-blocking at fixture scale. [gui][core]

## UX-B · Structural workspace (after F2 lands)
Goal: the two-level canvas + unified tree from the vision.
1. Unified project tree with root node (project context in inspector), atlas → folders/sprites/
   Animations node; virtualized at fixture scale. [gui]
2. List ⇄ grid display modes for sprites (header toggle, view state). [gui]
3. Two-level canvas: Project level (atlas group-cards, health badges, auto-grid, hidden-atlas
   compaction + "N hidden" chip) ⇄ Atlas level (pages spatial, page tabs removed, double-click
   page = camera zoom); breadcrumbs; overview thumbnails via result-store mips [Δ A9]. [gui][core]
4. Atlas visibility eye as typed operation [Δ A4][F2]. [gui][core]
5. Non-default markers + revert on every inspector field; "modified only" filter — needs
   effective-value provenance [Δ A2][F2]. [gui][core]
6. Problems panel: persistent severity list, click → sprite+canvas+field; status-bar ⚠ count;
   failed pack ends in action buttons. Feeds on structured notices. [gui][core]
7. Status bar segments (clickable): pack state, pages+fill% (→ per-page breakdown popover),
   ⚠ count, zoom, contextual key hints. [gui]
8. OS drag&drop ingestion (files/folders onto window/tree/canvas). [engine: window drop
   callback issue+PR][gui]
9. Numeric field upgrade: math expressions + label scrubbing (one widget, used everywhere). [gui]
10. Multi-select mass ops as single transactions: batch settings edit, bulk rename
    (find/replace), create-animation-from-selection parity. [gui][F2]
11. Hybrid board positions: auto-grid default, drag → freeform ("Move atlas" undoable op,
    `workspace` section), Tidy up re-grids, new atlases auto-place [Δ A10][F2]. [gui][core]
12. Canvas notes (post-structure, may slip to its own mini-epic): text stickers attached to
    atlas/canvas point; model objects (undo/history/tree/inspector), agent-readable via
    MCP [Δ A10][F2]. Arrows/rects drawing layer: far horizon, not planned. [gui][core]

## UX-C · Export rework [Δ A1, A7, A8]
Goal: "rules on project, participation on atlas", preflight-only modal.
1. Core: project-level target records + per-atlas participation, effective-target resolution,
   migration lifting identical per-atlas targets [core][F2].
2. Inspector: project root shows rules editor; atlas shows participation block (on/off + path
   override). One editing surface. [gui]
3. Preflight modal: scope = selection (N atlases / all), dry-run file list, overwrites amber,
   per-target degradation notices, Enter=run, Copy CLI command. [gui]
4. Export as ZIP: canonical layout {atlas}/{target}/ + manifest.json; CLI `--archive`. [core][gui]
5. Path templates {atlas}/{page}/{scale_suffix} in target paths + validation. [core]

## UX-D · Command palette & ops surface [F2, Δ A5]
Goal: every operation searchable and parameterized.
1. Palette (Ctrl+Shift+P): fuzzy over op registry + navigation ("go to sprite"), keybindings
   shown inline, frecency ranking, no-dead-ends fallback to sprite search. [gui]
2. Named undo everywhere: history labels from operation label templates; Edit menu shows
   "Undo: set padding 4 (3 sprites)". [gui][F2]
3. History panel v1 (list of transactions, click → affected object). [gui][F2]

## UX-E · Agent presence (with Epic A / MCP)
Goal: visible co-worker, no confirmations.
1. Presence chip (connected controller identity) + Disconnect/Revoke. [gui]
2. Live activity banner: current transaction description + View jump. [gui]
3. History authorship badges (AI/ME) from transaction author [Δ A6]. [gui]
4. Agent-touched highlight on tree rows / canvas regions (recent-transaction scope). [gui]

## UX-F · Polish pass (continuous, after B)
Empty states per panel; canvas backdrop options (checker/black/white/gray); micro-transitions
(state changes, palette open, level zoom); severity language audit; "Little Big Updates"
release-notes cadence. [gui]
