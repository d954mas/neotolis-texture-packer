# UX/UI Audit — ntpacker-gui (2026-07-10)

Point-in-time audit of the GUI at ~HEAD f671a2d + the in-flight canvas/toolbar
pass. Method: first-time-user and power-user code walkthroughs of
`apps/gui/*`, Nielsen heuristic pass, competitor gap pass against
`docs/research/*`. Excludes items already in flight at audit time (Cyrillic
font, compact toolbar, checkerboard, overlay toggles, scissor clip,
drag-threshold pan, pack stats, vertex readout) and the queued settings-panel
packet (region F + §3.3f). Line numbers in `main.c` drift (file under active
edit) — findings anchor on function names.

## Summary

The skeleton is sound: one fixed window, a canvas that renders the REAL packed
page with exact D4/hull/trim/pivot overlays and two-way selection sync, a
correct snapshot undo/dirty model, disciplined inline rename and context menus
sharing code paths. But the app is designed as if every operation were instant
and every message fit one line. Systemic weakness #1: **no asynchrony and no
persistent feedback surface** — pack/export run synchronously on the UI thread
(window freezes on big atlases, no progress, no cancel), and ALL feedback
squeezes through a single self-overwriting 256-byte status line; the notices
panel contract (principle 5, region H) is unimplemented. Systemic weakness #2:
the left panel does not scale — no search/filter, no folder collapse, no
multi-select, no keyboard navigation — so 10 atlases × 1000 sprites is
unworkable. Plus one real data-loss bug: File → Open discards unsaved changes
without confirmation.

## Critical

- **C1. Pack/Export synchronous on the UI thread — freeze, no progress, no
  cancel.** `do_pack` → blocking `tp_pack` (`gui_pack.c:321`); `do_export`
  loops blocking `tp_export_run` (`gui_pack.c:416`). `nt_ui_progress` unused.
  NFP/Minkowski packing takes seconds on 1000–2000 sprites; Export All repacks
  per target. Contradicts ux.md §3 (worker thread + progress). Fix: worker
  thread + `nt_ui_progress`; also the foundation for debounced auto-pack.
- **C2. No notices panel — all feedback via one overwritten status line.**
  Single `s_status[256]`; export notices are computed then freed
  (`tp_export_notices_free`, `gui_pack.c:433`) with only a
  "(metadata notices raised)" suffix shown; errors vanish on the next status
  write. Violates principle 5 / region H. Fix: implement region H — persistent
  severity-tagged notice list, click → focus offending sprite/target; errors
  persist until dismissed.
- **C3. File → Open / Ctrl+O silently discards unsaved changes.** `do_open`
  skips the `gui_project_is_dirty()` confirm that `request_new`/`request_exit`
  use. Data loss, separate from the known engine X-button gap. Fix: route Open
  through the same confirm path (AFTER_OPEN pending action). *(Dispatched to
  the in-flight GUI pass at audit time.)*
- **C4. Left panel does not scale — no filter, no folder collapse, no
  multi-select.** `build_rows` always fully expands every folder (no expanded
  bit/chevron); no search field (region D "🔍" unimplemented); selection is
  single-only (`s_sel_src`/`s_sel_child`) with no Ctrl/Shift-click; the atlas
  list is an unscrolled loop. Multi-select is a direct prerequisite for
  "Create animation from selection" (§3.7b). Fix: filter field, folder
  collapse, selection set with Ctrl/Shift, scrollable atlas list.

## Important

- **I1. Export targets cannot be created/configured in the GUI (region G
  unimplemented).** No target wrappers in `gui_project`; `tp_project_create`
  seeds `atlas1` with zero targets (`tp_project.c:301-310`), so a pure-GUI
  project always exports "Nothing to export"; only hand-editing the project
  file adds targets. Fix: region G UI (add/enable/exporter-id/out-path) or at
  least seed default `.ntpack` + `json-neotolis` targets. → folded into the
  settings-panel packet.
- **I2. No keyboard navigation in lists / no focus model.** Only global Ctrl
  chords + F5 in `handle_shortcuts`; no Up/Down/Home/End/Enter/Tab.
- **I3. Undo/redo nukes previews and selection across ALL atlases.**
  `do_undo`/`do_redo` call `gui_pack_clear(-1)` + `reset_selection`;
  `restore_from_buffer` unconditionally sets `preview_stale=true`
  (`gui_project.c:271`) even when undo restores the exact last-packed state.
  Fix: clear only the affected atlas, preserve selection, recompute stale by
  comparing against the last-packed snapshot.
- **I4. No session/window persistence, no Open Recent.** Fixed 1280×800
  window; UI-scale not persisted (TODO in `scale_item`); zoom/pan/selection
  reset every launch. Fix: app settings file (not project): window geometry,
  UI scale, recent projects, last project.
- **I5. Refresh (F5) is O(n²) and synchronous.** `do_refresh` double-scans all
  sources with linear `fp_find` per entry (~10⁸ strcmp at 10×1000 files); scan
  cache capped at 32 dirs (`gui_scan.c:17`). Fix: hash-table diff, off-thread;
  adaptive cache size.

## Polish

- "Needs action" accent color is blue (`0x4662A0`), same family as selection —
  comments claim amber; stale Pack button doesn't stand out. Use a distinct
  warning hue.
- Empty-project canvas hint says "press Pack (Ctrl+P)" with zero sources;
  §3.1 wants "Add a folder to start" + inline add button.
- Stale watermark incomplete vs §3.3b: chip + dimmed stats only; the canvas
  itself is not dimmed and has no corner "outdated" tag.
- No auto-pack toggle (region B / §3.2, default on).
- Stale docs: `gui_project.h` still claims in-process packing is blocked
  (#282) and stale never clears — code says otherwise.
- Silent row truncation at `MAX_ROWS 4096` (no notice).
- Tooltips only on Pack/Export/Refresh and truncated rows; none on menu items,
  zoom/page/overlay buttons, +Files/+Folder/+Atlas, `[x]`.
- Pack/Export/Refresh missing from the menubar (an "Atlas" menu would fix
  discoverability; mouse-complete is technically covered by the strip).

## Competitor gaps (not on the roadmap)

- "Show in Explorer" / open source PNG from a sprite row (TexturePacker).
- Sort/filter sprite list by attributes — rotated/trimmed/size (TexturePacker).
- Alias/duplicate indicator in the list — model already carries `alias_of`;
  GUI never shows it (TexturePacker, libGDX).
- Bulk rename / find-replace on names (TexturePacker, Unity).
- Copy region name/coords to clipboard (TexturePacker).
- Double-click row → zoom-to/center the region on canvas (TexturePacker,
  gdx-tpgui).

## What is already right (keep)

1. Canvas = the real packed artifact with honest D4/hull/trim/pivot overlays
   and two-way row↔region selection sync — inspection better than most free
   packers.
2. Snapshot undo/redo with memcmp dedup, gesture coalescing, byte-budget ring;
   `project_dirty` recomputed against the saved snapshot (clears ● on undo
   back to saved state).
3. `apply_pending` deferred-effects model: commits/dialogs/mutations run
   cleanly between frames; rename UX (seeded, Enter/Esc/click-outside) is
   desktop-correct.
4. Virtualized sprite vlist with per-row id recycling and provisioned state
   pool — the one place scale was designed through.
5. Mouse-complete parity where implemented: context menus share code paths
   with buttons/hotkeys; DPI-seeded UI scale; ellipsis + full-text tooltip.
