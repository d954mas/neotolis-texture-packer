# ntpacker — UX Vision & План

*2026-07-13 · синтез: ревью текущего GUI (свежие скриншоты + сверка кода) + разбор конкурентов (~150 источников) + библиотека паттернов любимых приложений (29 паттернов, Figma/Linear/Blender/VS Code/Aseprite/Godot/Raycast). Полные отчёты: [`../research/competitors-ux-2026-07.md`](../research/competitors-ux-2026-07.md), [`../research/ux-patterns-2026-07.md`](../research/ux-patterns-2026-07.md).*

---

## 1. Резюме

Ниша открыта шире, чем казалось. Каждый бесплатный GUI-пакер мёртв или умирает (free-tex-packer умер в 2021, домен запаркован в 2026; ShoeBox на трупе Adobe AIR; gdx-gui не отвечает на issues). TexturePacker любят за «just works» и ненавидят за три вещи: красные спрайты в бесплатной версии, отдельную CI-подписку и layout churn (перепаковка ломает ссылки и гадит в VCS). При этом **никто в мире** не закрывает: детерминированный headless-пайплайн, честную инспекцию того, что реально получит движок, десятки атласов как один проект, AI-агентов как операторов. Всё это — наши архитектурные сильные стороны (typed operations, канонические IR, MCP в спеке).

> **Ревизия 13.07 (обсуждение с владельцем):** специальные режимы канваса Explain/Diff и карточка «why here» отклонены как усложнение. Детерминизм остаётся свойством ядра без UI; «стеклянный корпус» строится честными поверхностями (problems, preflight, деградация), а не режимами. Масштаб — инвариант, а не режим.

**Позиционирование:** *«Пакер со стеклянным корпусом»* — первый пакер, которому видно всё и которому поэтому доверяют: люди и агенты работают в одной живой сессии, каждая операция именована, обратима и объяснима.

## 2. Ревью текущего состояния

Скриншоты снимались dev-режимом `--shot` (воспроизведение: `ntpacker-gui --shot=out.png --size=1920x1080 examples/showcase/showcase.ntpacker_project`, плюс `--shot-stale` и пустой проект). Аудит от 10.07 устарел наполовину — редизайн-пакеты внедрены.

**Уже сильно (сохранить и углублять):**
- Канвас показывает **настоящую** упакованную страницу с честными D4/hull/trim/pivot оверлеями и двусторонней синхронизацией выделения — ни один бесплатный пакер так не умеет.
- **Превью экспорт-таргета с чипом деградации** (что именно потеряет конкретный экспортёр) — редкая фича, у TP такого нет.
- Асинхронный Pack/Export с Cancel; amber-стейл система (кнопка + затемнение + чип); иконки/иерархия/тултипы; снапшотный undo; empty state с CTA; экспорт-таргеты в GUI с дефолтом.

**Открытые дыры (ядро плана):**

| Дыра | Тяжесть |
|---|---|
| Нет drag&drop из ОС — главный жест добавления отсутствует | критично |
| Нет поиска/фильтра по спрайтам; папки не сворачиваются; список атласов без скролла | критично при 10+ атласах |
| Нет клавиатурной навигации по спискам, нет фокус-модели | важно |
| Notices: одно сообщение, ошибки перетираются; problems-панели нет | важно |
| Нет персистентности сессии: геометрия, зум, Open Recent | важно |
| Undo сбрасывает превью+выделение всех атласов; шаги безымянные | важно |
| Refresh синхронный O(n²), кэш 32 папки | важно на масштабе |
| Нет: command palette, sort, bulk rename, show-in-explorer, copy name, double-click→zoom | накопленный сор |

## 3. Рынок: за что любят и ненавидят

**Table stakes (без этого не воспринимают всерьёз):** smart folders + watch; живой явный результат упаковки (успех/провал громко); GUI = CLI = один проектный файл + no-op rebuild; pivot и 9-slice редакторы с ручками (главная причина платить за TP); engine-exact экспортёры (никакого «sed по JSON»); crash-safe сессии, относительные пути; DPI/контраст/тултипы (главная жалоба на rTexPacker — кастомный UI судят именно по этому).

**Топ-жалобы на TP (наши бесплатные победы):** красные спрайты + некоммерческая лицензия (родили всю экосистему альтернатив); CI-лицензирование с подпиской и падающим активационным сервером; layout churn (признано вендором, чинилось только в TP7) + SmartUpdate-хэши в диффах.

**Незанятые ниши (проверено — никто не делает):**
1. **Детерминированная, инкрементальная, диффуемая упаковка** — одна причина закрывает 4 боли: VCS-шум, CI-воспроизводимость, размер live-ops патчей, batch-время.
2. **Observability**: waste heatmap, «почему спрайт здесь», diff раскладки между паками. Единственный prior art — внутренний debug-SVG WebRender'а.
3. **AI/MCP-пайплайн**: у Aseprite 6+ MCP-серверов, у атлас-пакинга — два игрушечных. Окно открыто сейчас.
4. **Мульти-атласный менеджер масштаба**: наследование настроек, инкрементальные пересборки. У TP — shell-циклы, у Unity — многочасовые пересборки мира.

## 4. Vision: три кита + полировка

**Сквозной принцип — один UX на любой масштаб** *(решение владельца 13.07)*: маленький и большой проект получают одни и те же фичи с самого начала. Каждый список виртуализирован и фильтруем, каждая операция работает на мульти-выделении (одной транзакцией = одним undo), настройки наследуются (проект → атлас → спрайт, с маркером «переопределено здесь»), долгие операции всегда с прогрессом и Cancel. Никакого «режима большого проекта».

**Каркас канваса — два уровня** *(решение владельца 13.07)*: **Project** (все атласы) ⇄ **Atlas** (рабочий уровень: атлас со ВСЕМИ его страницами, разложенными рядом). Третьего уровня «страница» нет — page-табы уходят: страницы пространственные, double-click по странице = зум камеры к ней (движение камеры, не смена режима), дальше зум до текселей; спрайты выбираются прямо на этом уровне, синхронизация с деревом/инспектором обычная. Навигация: double-click по группе спускает, хлебные крошки в тулбаре (Project ▸ animals) поднимают.
Почему не один сплошной уровень (всё на одном пространстве, только зум): контент канваса перегенерируется каждым Pack — размеры и раскладка страниц меняются, единое пространство «плывёт» под ногами и дезориентирует; fit и контекст инспектора становятся неоднозначными («fit чего?»); на 30 атласах × страницы нужен агрессивный VRAM-куллинг. Дискретная граница атласа даёт стабильную локальную систему координат и однозначный контекст — и это ровно модель Figma: внутри страницы — сплошное пространство, между страницами — дискретный список.
На уровне Project атлас — **группа-карточка**: имя, бейджи здоровья (fill% / outdated / not packed / ⚠), глаз видимости — и страницы внутри как живые миниатюры последнего пака. Страницы разных атласов не смешиваются в потоке: в строке соседствуют только целые группы; очень большой атлас показывает первые N страниц + «+N». Batch-действия («Pack stale (N)», «Export all», прогресс на группах) — на уровне Project.
Страницы — вывод пакера, руками не редактируются: манипуляции (V2 — drag спрайтов/папок) идут между атласами, не между страницами одного атласа. **Позиции досок — freeform без auto-reflow** *(финальная ревизия 13.07: «сжатие сетки» удалено как пережиток)*: у каждой доски всегда есть позиция (данные проекта, секция `workspace`); при создании атласа она проставляется автоматически в свободное место — дальше доска твоя. **Тул никогда не двигает доски сам**: скрыл атлас — место пустует, никто не переезжает (чип «N hidden» возвращает на место); атлас вырос — растёт своя доска (страницы внутри переносятся рядами, у огромных — плитка «+N pages»), соседей не раздвигает, при коллизии — перекрытие; уменьшился — место остаётся. Перекрытие легально (z-order: последняя тронутая сверху), но **не семантично**: дроп атласа на атлас ничего не делает с данными — семантические дропы на доску только у заметок (парентинг) и V2-переноса спрайтов; слияние атласов — только явной командой, если появится. Перенос — undoable «Move atlas»; **Tidy up** — единственное авто-перемещение, и его вызываешь ты.
**Текстовые объекты на канвасе** *(идея владельца 13.07, реализация после структуры)*: текст, привязанный к атласу или точке канваса — объект модели (undoable, в истории, виден в дереве/инспекторе), а не рисунок. **В первую очередь — инструмент владельца** (пометки, заголовки секций, TODO); читаемость агентом (контент + цвет как токен: «разбери атласы с красными пометками») — бонусное свойство. **Стиль — на объект целиком** (Figma-lite): размер S/M/L/XL, болд, курсив (нужны bold/italic-начертания шрифта — ассет-работа на стороне движка), цвет текста и фона из палитры свотчей (10–12; фон «none» = надпись, с фоном = стикер), выравнивание. Отложено, не отвергнуто: произвольный цвет (нужен color-picker-виджет в nt_ui), смешанные стили внутри одного текста (rich-text ранов v1 не делаем). **Где живут** *(владелец 13.07)*: один тип объекта, **принадлежность — автопарентинг по месту** (идиома Figma, без выбора «вида» при создании): положил на доску атласа → заметка атласа (едет с доской, скрывается с глазом, удаляется с атласом — confirm называет число заметок, undo возвращает); положил на пустой канвас → заметка проекта; перепарентовка — перетаскиванием (цель подсвечивается). В дереве — свёрнутая группа «Notes (N)» внутри атласа (как узел Animations) или под корнем, клик → зум к заметке; редактирование — в инспекторе; Ctrl+F ищет по тексту заметок; на канвасе — оверлей-тоггл «show notes» (состояние вида), отдельного режима нет. V1: заметки живут на уровне Project (внутри атласа якорить не к чему — пак перегенерируется). Свободные заметки в дереве — группой «Notes» под корнем после атласов; вложенные — внутри своего атласа; перепарентовка на канвасе двигает их и в дереве. **Идея на будущее (записана, не в плане):** заметки на спрайте — как пометки на объектах модели (бейдж в дереве/инспекторе), не как объекты канваса. Полный drawing-слой (стрелки, ректы, свободное рисование) — отклонён на далёкий горизонт: дорого (трансформ/z-order/текст на канвасе) и обслуживает разговор о работе, а не работу. **Решение владельца:** глаз — полноценная модельная операция: именованная запись в истории («Hide atlas ui»), undo/redo, сохраняется в проекте, помечает dirty. Наш движок вывезет всё это дёшево (рендерер игрового движка, LRU результатов уже в спеке) — Qt-конкурентам такое дорого.

**Левая панель — единое дерево проекта** *(решение владельца 13.07)*: вместо раздельных секций ATLASES / SPRITES / ANIMATIONS — один tree с **корневым узлом-проектом** (клик по корню → настройки проекта в инспекторе: дефолты атласов + экспорт-правила; проект — осязаемый объект, а не скрытый режим): атлас раскрывается в свои папки, спрайты и узел Animations. Структура панели зеркалит канвас (Project → Atlas), выделение синхронно в обе стороны; фильтр Ctrl+F ищет по всему проекту и показывает совпадения под их атласами (в master-detail это невозможно); cross-atlas drag работает и в дереве. **Сортировка** — состояние вида, не данные. Четыре ключа × два направления (8 вариантов): имя (дефолт) / размер (площадь px) / дата изменения файла (mtime из ФС) / дата добавления в атлас (added_at); повторный клик по активному ключу переворачивает направление (стрелка ↑↓ у ключа). Плюс независимая галочка «⚠ наверх» — поднимает проблемные поверх любой сортировки, сохраняя порядок внутри групп. Ключ действует внутри каждого уровня дерева (файлы в папке, папки в атласе); атласы по умолчанию — в ручном порядке проекта, сортировка их не переписывает. «Своя логика порядка» = папки: файловая система — источник правды для smart-folder, её порядок не переопределяется (рукотворный порядок сгнил бы при rescan). Ручной порядок остаётся только там, где он семантика: кадры анимации, порядок атласов в проекте (undoable-операции).

**Время на ассетах** *(решение 13.07 — показывать, не «штамповать»)*: mtime файла берётся из ФС живьём (инспектор, тултип, ключ сортировки «recently modified») — в проект не пишется; дата добавления источника в атлас пишется один раз при добавлении и больше не меняется («recently added», бейдж new). Никаких постоянно обновляемых per-asset тайм-стампов в файле проекта — это ровно тот VCS-шум, за который мы ругаем TexturePacker (SmartUpdate-хэши). «Кто и когда менял в проекте» — это уже история транзакций, отдельного хранения не требует.

**Превью и карточки** *(решение владельца 13.07)*: миниатюры — дефолт, не режим: у каждой строки спрайта в дереве превью (async-кэш миниатюр), у строки атласа — мини-превью страницы; hover по строке → крупное превью 1:1. Секция SPRITES имеет два режима отображения — список ⇄ карточки (стандарт ассет-браузеров: Explorer/Unity/Godot/Figma), переключатель в заголовке секции; режим — состояние вида панели. Карточному виду атласов отдельный режим не нужен — это и есть Overview.

**Кит 1 — Скорость до результата** *(владелец: wow-фактор №1)*
Золотой путь: запуск → перетащил папку → атлас на экране с правильными дефолтами → Export. Меньше 60 секунд, ноль диалогов. Пустые состояния называют и мышиный путь, и клавишу. Каждое взаимодействие UI < 100мс; спиннеров нет для операций < 1с. Pack остаётся явным (решение владельца: автопака нет — упаковка может быть долгой), но стейл-система делает «правку → P → результат» двухтактовой.

**Кит 2 — Стеклянный корпус (glass box)**
Только честные поверхности — специальных режимов канваса нет (Explain/Diff и «why here» отклонены 13.07):
- **Уже есть, углубляем**: честный канвас (реальная страница + реальные concave-оверлеи) и превью экспорт-таргета с чипом деградации — фундамент доверия.
- **Problems-панель** (VS Code): все диагностики упаковки/экспорта агрегированы, живут до починки, клик = спрайт в дереве + зум канваса + подсветка поля в инспекторе. Провал упаковки заканчивается кнопками действий, не голой ошибкой.
- **Preflight экспорта**: dry-run список файлов, перезаписи амбером, потери по таргетам — до записи на диск.
- **Именованный undo**: «Undo: set padding 4 (3 sprites)» — семантические транзакции из master-spec это уже дают.
- **Non-default маркеры** (Godot): цветная полоска + revert на каждом поле, отличающемся от дефолта/унаследованного; фильтр «только изменённые».
- **Fill по страницам**: клик по «fill 87%» в статус-баре → разбивка (страница 1 — 94%, страница 2 — 31%) — видно, где резерв, без всяких heatmap.
- **Детерминизм — свойство ядра, без UI**: тот же вход → байт-в-байт тот же выход (CI-кэш, no-op rebuild, воспроизводимость багов). «Стабильная упаковка» для live-ops-патчей — далёкий горизонт, опционально, по запросу.

**Кит 3 — Двухвидовой инструмент (человек + агент)**
Решения владельца: видно, что агент работает и что делает; подтверждений на действие НЕТ (allow-all/deny по спеке); история показывает авторство.
- **Presence-чип** в шапке: «⚡ Claude — connected», активность в реальном времени.
- **Activity feed**: живая лента транзакций агента (те же именованные транзакции), кликабельная — прыжок к затронутому объекту.
- **История с авторством**: каждая запись — кто (человек/агент), что, когда; Undo работает одинаково для обоих.
- MCP/CLI-parity уже в архитектуре — GUI просто обязан её показывать.

**Поперечный слой — полировка** *(владелец: wow-фактор №2)*
Linear/Figma-ощущение в нативном инструменте: command palette (Ctrl+Shift+P) индексирует ВСЕ операции включая параметризованные («set max page size 1024») — typed operation engine, который сейчас строится, это буквально реестр для палитры; каждый тултип/меню печатает свой шорткат; Figma-грамматика канваса (Ctrl+Scroll zoom-at-cursor, Space-pan, F = zoom-to-selection, Shift+0 = 1:1); одноклавишные глаголы на выделении (T trim, R rename, F focus, Del); математика в числовых полях (`2048/2`, `+8`) и скраббинг за лейбл; статус-бар из кликабельных сегментов («3 pages · 87% fill · ⚠ 2»); микроанимации на изменение состояния.

**Модель экспорта** *(обсуждение 13.07)*. Дефекты текущей схемы: таргеты только на атласе (30 атласов = 30 одинаковых конфигов), две поверхности редактирования (панель + модалка Ctrl+E), превью деградации оторвано от таргетов. Новая модель:
1. **Правила — на проекте, участие — на атласе** *(уточнено после обсуждения)*: таргеты объявляются и редактируются ТОЛЬКО на проекте (корневой узел дерева: «Defold → `assets/atlases/{atlas}/`»). На атласе — не редактор, а компактный блок участия: галочка вкл/выкл в каждом таргете + опционально переопределение пути. Никакого дублирования списка таргетов на атласе.
2. **Ctrl+E ничего не редактирует** — это только preflight (dry-run список файлов, перезаписи амбером, деградации, Copy CLI, Enter). «Повторить последний экспорт» = Ctrl+E → Enter.
3. **Скоуп = текущее выделение** (инвариант мульти-выделения): выделил 2 атласа → «Export 2 atlases»; ничего не выделено на уровне Project → «Export all»; внутри атласа → этот атлас. Preflight всегда называет скоуп явно.
4. **Анатомия таргета — три оси**: формат данных (экспортёр из реестра); изображение (PNG дефолт, WebP опция, квантизация pngquant-style — у TP платно, у нас бесплатно; premultiply по капабилити экспортёра; GPU-форматы KTX2/ASTC — далёкий горизонт, Defold сжимает сам через texture profiles); скейл-варианты (список factor+suffix, Figma-модель; каждый скейл — отдельный пак с масштабированием ДО упаковки; для Defold v1 не критично — слот в модели закладываем сейчас). Пути — шаблонами: `{atlas}`, `{page}`, `{scale}`/`{scale_suffix}`.
5. **Export as ZIP** *(идея владельца 13.07)*: обёртка доставки, не вид таргета — опция в preflight; те же скоуп и dry-run, назначение — один архив с канонической структурой `{atlas}/{target}/…` + `manifest.json` (файлы, проект, дата, версия тула); персональные пути таргетов в архиве игнорируются. Staging уже атомарный → фича почти бесплатна. Прецедент: Figma сама отдаёт zip при массовом экспорте.
6. Соседняя фича на горизонт (не v1): **Package project** — передать редактируемый проект (файл + исходники, пути переписаны в относительные), как InDesign Package / AE Collect Files. Это «отправить дизайнеру проект», а архив выше — «отправить результаты».
Прецеденты: Figma (пресеты scale+format+suffix на узле — эталон предсказуемости), Aseprite (запомненный «repeat last export»), антипримеры — TexturePacker (один формат на .tps, второй движок = дублируй проект, скейлы платно) и Photoshop Export As (разовый, без памяти).

## 6. UX-метрики (регрессионные бюджеты)

Калибровка масштаба *(владелец, 13.07)*: реальные игры — **30+ атласов, 5000+ спрайтов**. Все бюджеты проверяются на бенч-фикстуре этого размера (эпик UX-G0); производительность списков, refresh и кэш миниатюр — приоритет первой волны, не «потом на масштабе».

- Любое взаимодействие UI: **< 100мс** (включая фильтр по 5000 строк); спиннер только после 1с.
- Золотой путь «папка → экспорт»: **< 60с, 0 диалогов**.
- «Re-export после правки исходника»: **≤ 2 действия** (P → Ctrl+E).
- «Найти спрайт по имени из 5000»: **≤ 3с** (Ctrl+F или палитра).
- Уровней progressive disclosure в инспекторе: **≤ 2**.
